# 核心算法/核心机制

## 目标
深入分析最复杂、最核心的机制，理解系统的"心脏"。

## 分析要求

1. 识别最值得深入的 3~5 个文件/类/函数
2. 解释它们的算法、状态机或调度逻辑
3. 说明它们为什么复杂，复杂性来自哪里
4. 说明这些机制对整体架构的作用
5. 给出简化版伪代码或流程图

# 扩展点/插件机制

## 目标
分析系统能不能"长大"，理解扩展机制。

## 分析要求

1. 找出插件、hook、callback、registry、middleware、adapter、strategy 等扩展点
2. 说明新增功能应该接入哪里
3. 分析扩展机制是否稳定、是否易测试
4. 指出哪些地方是硬编码，限制了扩展
5. 说明系统更像"可插拔平台"还是"固定流程程序"

## 六、反模式（不要这样做）

复刻时**容易犯**的错误，作者刻意避开了：

❌ **直接拼字符串**：`exec(\`cmd ${userInput}\`)` 永远是错的
❌ **shell: true 默认开**：让 shell 元字符成为攻击面
❌ **正则单趟剥离**：必须不动点迭代（见 5.2）
❌ **静默降级安全设置**：用户开了沙箱但 deps 缺失 → 必须告知
❌ **整串字符串当原子单位做安全决策**：`startsWith("git")` 拦不住 `ls && git ...`
❌ **<根据具体代码补充>**

## 模板使用要点

1. **目录跟随实际章节** —— 删了哪一节，目录里也删
2. **行数控制**：
   - 小型文件夹（< 1500 行源码）→ detail.md ≤ 500 行
   - 中型 → 500–1000 行
   - 大型 → 1000–2000 行
   - **超过 2500 行的 detail.md 多半在掺水**，回去精简
3. **每章必须有具体代码定位**（[file:line](#L) 链接）
4. **代码块控制在 15 行以内**，前后用注释说明省略了什么
5. **作者血泪必转**：注释里的 issue 编号、HackerOne 引用、CVE、攻击 trace、PR 链接 → 全部搬运到 detail.md，原样保留 ID 和上下文

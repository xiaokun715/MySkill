## 模板使用要点

1. **目录跟随实际章节** —— 删了哪一节，目录里也删
2. **"逐文件深度解析"节不可省略** —— 即使用户说"只要快速概览"也不省略（那是 /init 的事）
3. **行数控制**：
   - 小型文件夹（< 500 行源码）→ detail.md ≤ 300 行
   - 中小型（500–2000 行）→ 500–800 行
   - 中型（2000–5000 行）→ 800–1500 行
   - 大型（5000–10000 行）→ 1500–2500 行
   - 超大型（> 10000 行）→ 2500+ 行，上不封顶
4. **每章必须有具体代码定位**（[file:line](#L) 链接）
5. **代码块控制在 15 行以内**，前后用注释说明省略了什么
6. **作者血泪必转**：注释里的 issue 编号、HackerOne 引用、CVE、攻击 trace、PR 链接 → 全部搬运到 detail.md，原样保留 ID 和上下文
